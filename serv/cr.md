## 🔹 Базовые метрики звонков

### <a id="head-1"></a>📥 # входящих

**📝 Описание:**  
Количество входящих звонков за выбранный период

> 📌 **Примечание:** Учитываются все звонки, поступившие на номера клиники, независимо от результата обработки
>
> 🔗 **Источник данных:** Телефония / колл-трекинг / CRM

---

### <a id="head-2"></a>❌ # пропущенных

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Пропущенные вызовы |
| Missed Calls |

**📝 Описание:**  
Количество входящих звонков, пропущенных оператором (не обработанных)

> 📌 **Критерий:** Звонок считается пропущенным, если не был принят оператором в течение установленного времени ожидания
>
> 🎯 **Интерпретация:** Ключевой показатель нагрузки на кол-центр и качества распределения звонков

---

## 🔹 Длительность и время ожидания

### <a id="head-3"></a>⏱️ Средняя длительность входящего

**📋 Альтернативные наименования:**

| Название | 
|----------|
| AHT (Average Handle Time) |
| Среднее время разговора |

**📝 Описание:**  
Среднее количество секунд длительности звонка входящего

> 🧮 **Расчёт:** `Сумма длительности входящих звонков / # входящих`
>
> 🎯 **Интерпретация:** 
> - Низкое значение: возможная спешка, неполная консультация
> - Высокое значение: сложные запросы или неэффективный скрипт
> - Оптимум зависит от специфики клиники и типа звонка

---

### <a id="head-4"></a>⏳ Средняя длительность ожидания входящего

**📋 Альтернативные наименования:**

| Название | 
|----------|
| ASA (Average Speed of Answer) |
| Среднее время в очереди |

**📝 Описание:**  
Среднее количество секунд ожидания звонка входящего до ответа оператора

> 🧮 **Расчёт:** `Сумма длительности ожидания входящих звонков / # входящих`
>
> ⚠️ **Важно:** В расчёт могут включаться только звонки, дошедшие до оператора, или все входящие — уточните логику в настройках
>
> 🎯 **Интерпретация:** 
> - < 30 сек: отличный показатель
> - 30–60 сек: допустимый уровень
> - > 60 сек: риск потери клиентов, требуется оптимизация

## 🔹 Динамика (YTY / MTM)

### <a id="head-5"></a>📅 YTY

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Year-To-Year |
| Изменение к прошлому году |

**📝 Описание:**  
Year-To-Year. Абсолютное изменение показателя к аналогичному периоду прошлого года

> 🧮 **Расчёт:** `Значение текущего периода − Значение аналогичного периода прошлого года`
>
> 📌 **Пример:** Если в январе 2026 было 1200 звонков, а в январе 2025 — 1000, то YTY = +200

---

### <a id="head-6"></a>📈 YTY (%)

**📋 Альтернативные наименования:**

| Название | 
|----------|
| YoY Growth |
| Темп роста к прошлому году |

**📝 Описание:**  
Year-To-Year. Процентное изменение показателя к аналогичному периоду прошлого года

> 🧮 **Расчёт:** `100 × (Значение текущего периода − Значение прошлого года) / Значение прошлого года`
>
> 🎯 **Интерпретация:** 
> - Положительное значение: рост показателя
> - Отрицательное значение: снижение
> - Позволяет оценить сезонность и долгосрочные тренды

---

### <a id="head-7"></a>🗓️ MTM

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Month-To-Month |
| Изменение к прошлому месяцу |

**📝 Описание:**  
Month-To-Month. Абсолютное изменение показателя к аналогичному периоду прошлого месяца

> 🧮 **Расчёт:** `Значение текущего периода − Значение аналогичного периода прошлого месяца`
>
> ⚠️ **Важно:** При сравнении месяцев с разным количеством дней интерпретируйте с осторожностью

---

### <a id="head-8"></a>📊 MTM (%)

**📋 Альтернативные наименования:**

| Название | 
|----------|
| MoM Growth |
| Темп роста к прошлому месяцу |

**📝 Описание:**  
Month-To-Month. Процентное изменение показателя к аналогичному периоду прошлого месяца

> 🧮 **Расчёт:** `100 × (Значение текущего периода − Значение прошлого месяца) / Значение прошлого месяца`
>
> 🎯 **Интерпретация:** Оперативный индикатор для ежемесячного контроля: реакция на рекламные активности, изменения в расписании, сезонные всплески

## 🔹 Средние значения по дням

### <a id="head-9"></a>📉 Среднее кол-во пропущенных

**📝 Описание:**  
Среднее по дням количество пропущенных звонков оператором

> 🧮 **Расчёт:** `# пропущенных / количество дней в периоде`
>
> 🎯 **Интерпретация:** 
> - Помогает выявить «проблемные» дни (например, понедельники или предпраздничные дни)
> - Удобно для сравнения периодов разной длины

---

### <a id="head-10"></a>⏱️ Среднее время ожидания

**📝 Описание:**  
Среднее по дням время ожидания на линии

> 🧮 **Расчёт:** `Сумма времени ожидания в периоде / количество дней в периоде`
>
> ⚠️ **Важно:** Это среднее от ежедневных средних, а не общее время ожидания, делённое на дни
>
> 🎯 **Интерпретация:** Позволяет оценить стабильность качества обслуживания в динамике

---

### <a id="head-11"></a>📥 Среднее кол-во входящих

**📝 Описание:**  
Среднее по дням количество входящих звонков оператором

> 🧮 **Расчёт:** `# входящих / количество дней в периоде`
>
> 🎯 **Интерпретация:** 
> - Базовый индикатор нагрузки на кол-центр
> - Используется для планирования графиков операторов и оценки потребности в ресурсах

| Показатель | Ед. измерения | Категория | Формула | Доступно в разрезе |
|------------|---------------|-----------|---------|-------------------|
| # входящих | шт | Объём | — | Дата, Канал, Отделение, Оператор |
| # пропущенных | шт | Качество | — | Дата, Канал, Отделение, Оператор |
| Средняя длительность входящего | сек | Эффективность | Σдлительности / # входящих | Дата, Канал, Оператор |
| Средняя длительность ожидания | сек | Качество | Σожидания / # входящих | Дата, Канал, Оператор |
| YTY / YTY (%) | шт / % | Динамика | Δ к прошлому году | Период, Канал |
| MTM / MTM (%) | шт / % | Динамика | Δ к прошлому месяцу | Период, Канал |
| Среднее кол-во пропущенных | шт/день | Нагрузка | # пропущенных / дни | Отделение, Оператор |
| Среднее время ожидания | сек/день | Качество | Σожидания / дни | Отделение, Оператор |
| Среднее кол-во входящих | шт/день | Нагрузка | # входящих / дни | Отделение, Оператор |

---

> 💡 **Рекомендации по использованию:**
> - **Процент пропущенных**: Рассчитывайте как `# пропущенных / # входящих × 100` — целевое значение < 5–10%
> - **Время ожидания**: Мониторьте в реальном времени для оперативного перераспределения операторов
> - **YTY vs MTM**: YTY лучше показывает тренды, MTM — оперативные изменения; используйте оба для полной картины
> - **Средние по дням**: Удобны для сравнения периодов разной длины, но скрывают внутридневную динамику — уточняйте детали при аномалиях
> - **Длительность разговора**: Анализируйте в связке с конверсией в запись — короткий разговор не всегда означает плохое качество

---

> 🔐 **Требования к данным:**
> - Интеграция с телефонией/колл-трекингом с передачей метаданных: время начала/окончания, статус ответа, идентификатор оператора
> - Корректная разметка «пропущенных» звонков (логика определения должна быть единой для всех источников)
> - Настройка справочника рабочих дней для корректного расчёта средних значений
> - Синхронизация временных зон между системами (особенно при работе с филиалами в разных регионах)
