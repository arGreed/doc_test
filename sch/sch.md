## 🗓️ Базовые метрики расписания

### <a id="head-1"></a>🔄 # смен

**📝 Описание:**  
Количество смен в выбранном периоде

> 🧮 **Расчёт:** `Количество часов в выставленном расписании / 12`
>
> 📌 **Определение:** Под 1 сменой понимается 12 часов выставленного расписания

---

### <a id="head-2"></a>📝 # записей в расписании, шт (по дате создания записи)

**📝 Описание:**  
Количество записей, созданных в расписании за период (независимо от того, пришёл пациент или нет)

> 📌 **Особенность:** Считается по дате **создания** записи, а не по дате приёма

---

### <a id="head-3"></a>📋 # записей в расписании

**📝 Описание:**  
Количество записей, которые были сделаны в расписании (независимо от того, пришёл пациент или нет)

> 🧮 **Определение:** Каждая запись занимает один временной слот в расписании
>
> 📌 **Особенность:** Рассчитываются в момент прихода пациента

---

### <a id="head-4"></a>👤 # записавшихся

**📝 Описание:**  
Количество уникальных пациентов, записавшихся в расписание

> 📌 **Примечание:** Один пациент может быть записан на несколько слотов времени — учитывается как один уникальный пациент

---

## ✅ Доходимость и конверсия

### <a id="head-5"></a>✔️ # записей дошедших

**📝 Описание:**  
Количество записей, по которым пациенты фактически пришли на приём

> 📌 **Критерий:** Учитываются записи с отметкой «Пришёл» / «Дошёл» в МИС

---

### <a id="head-6"></a>📈 Записей дошедших, YTY%

**📝 Описание:**  
Процентное отклонение количества дошедших записей текущего года относительно аналогичного периода прошлого года

> 🧮 **Расчёт:** `100 * (# записей дошедших - # записей дошедших (прошлый год)) / # записей дошедших (прошлый год)`

---

### <a id="head-7"></a>📊 Записей в расписании, YTY%

**📝 Описание:**  
Процентное отклонение общего количества записей в расписании текущего года относительно прошлого года

> 🧮 **Расчёт:** `100 * (# записей в расписании - # записей в расписании (прошлый год)) / # записей в расписании (прошлый год)`

---

### <a id="head-8"></a>🎯 CR — Конверсия из записи в приход (Conversion Rate)

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Доходимость |
| Конверсия записи |
| Show Rate |

**📝 Описание:**  
Доля пришедших пациентов от общего количества записавшихся в расписание

> 🧮 **Расчёт:** `# записей дошедших / # записей в расписании * 100`
>
> 🎯 **Интерпретация:** Ключевой показатель эффективности работы регистратуры и напоминаний о приёме

---

### <a id="head-9"></a>🔁 # перезаписи, шт

**📝 Описание:**  
Количество записей, сделанных после приёма пациента в тот же день (повторная запись в день визита)

> 📌 **Примечание:** Индикатор комплексного обслуживания или допродажи услуг

---

### <a id="head-10"></a>🔁 # перезаписи дошедшие, шт

**📝 Описание:**  
Количество дошедших перезаписей — записей, сделанных после приёма в тот же день, по которым пациент фактически вернулся

---

### <a id="head-11"></a>📈 Доходимость по перезаписям, %

**📝 Описание:**  
Доля дошедших перезаписей от общего количества перезаписей

> 🧮 **Расчёт:** `100 * # перезаписи дошедшие / # перезаписи`
>
> 🎯 **Интерпретация:** Показывает эффективность «горячих» допродаж в день приёма

---

## 🏢 Загрузка ресурсов

### <a id="head-12"></a>🏥 Загрузка помещений (кабинетов, операционных)

**📝 Описание:**  
Доля заполненных слотов расписания помещения к общему количеству доступных слотов

> 🧮 **Расчёт:** `# заполненных слотов по помещению / # выставленных слотов по помещению * 100`
>
> 🎯 **Интерпретация:** Показатель эффективности использования инфраструктуры

---

### <a id="head-13"></a>👨‍⚕️ Загрузка врачей пациентами, факт

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Загрузка врачей пациентами факт |
| Physician Utilization Rate |

**📝 Описание:**  
Доля заполненных слотов расписания дошедшими пациентами к общему количеству выставленных слотов

> 🧮 **Расчёт (вариант 1):** `# заполненных слотов дошедшими пациентами / # выставленных слотов * 100`
>
> 🧮 **Расчёт (вариант 2):** `100 * Факт часов пациентов / Факт часов врачей`
>
> ⚠️ **Важно:** Не учитываются врачи, по которым не ведётся расписание

---

### <a id="head-14"></a>👨‍⚕️ Загрузка врачей пациентами, план, %

**📝 Описание:**  
Плановая загрузка врачей пациентами, внесённая в план

> 🧮 **Расчёт:** `100 * План часов пациентов / План часов врачей`
>
> ⚠️ **Примечание:** Может присутствовать только на дашбордах с плановыми показателями

---

### <a id="head-15"></a>📉 Загрузка врачей, отклонение от плана, %

**📝 Описание:**  
Отклонение между фактической и плановой загрузкой врачей пациентами

> 🧮 **Расчёт:** `Загрузка врачей пациентами факт, % - Загрузка врачей пациентами план, %`
>
> ⚠️ **Примечание:** Может присутствовать только на дашбордах с плановыми показателями

---

### <a id="head-16"></a>✅ Исполнение плана по загрузке врачей, %

**📝 Описание:**  
Процентное отклонение фактической загрузки от плановой

> 🧮 **Расчёт:** `100 * (Факт часов пациентов / План часов врачей - 1)`
>
> ⚠️ **Примечание:** Может присутствовать только на дашбордах с плановыми показателями

---

### <a id="head-17"></a>📊 Загрузка расписания пациентами, %

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Загрузка в % |
| Schedule Utilization |

**📝 Описание:**  
Процентное отношение фактических часов приёма пациентов к плановым часам врачей

> 🧮 **Расчёт:** `100 * (Факт часов пациентов / План часов врачей)`

---

### <a id="head-18"></a>🏢 Загрузка центров врачами, факт / % покрытия мощности

**📋 Альтернативные наименования:**

| Название | 
|----------|
| % покрытия мощности |
| Capacity Coverage |

**📝 Описание:**  
Покрытие текущей мощности выставленным расписанием врачей

> 🧮 **Расчёт (вариант 1):** `100 * Факт часов врачей / Мощность`
>
> 🧮 **Расчёт (вариант 2):** `100 * Ёмкость выставленного расписания / Мощность`
>
> ⚠️ **Требования:** Требуется ведение расписания клиники. Метрика может иметь смысл до уровня отделений

---

### <a id="head-19"></a>🏢 Загрузка центров врачами, план, %

**📝 Описание:**  
Плановая загрузка центров (клиники) врачами

> 🧮 **Расчёт:** `100 * План часов врачей / Мощность`
>
> ⚠️ **Примечание:** Может присутствовать только на дашбордах с плановыми показателями

---

### <a id="head-20"></a>📉 Загрузка центров врачами, отклонение от плана, %

**📝 Описание:**  
Разница между фактическим и плановым использованием времени врачей

> 🧮 **Расчёт:** `Загрузка центров врачами факт, % - Загрузка центров врачами план, %`
>
> ⚠️ **Примечание:** Может присутствовать только на дашбордах с плановыми показателями

---

### <a id="head-21"></a>🏥 Загрузка клиники пациентами, факт

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Загрузка центров пациентами факт |
| Загрузка мощности |
| Загрузка клиники, % |
| Покрытие мощности кабинетов расписанием, % |

**📝 Описание:**

>
> 🧮 **Расчёт (вариант 2):** `Загрузка врачей * Покрытие мощности`

---

### <a id="head-22"></a>🏥 Загрузка клиники пациентами, план, %

**📝 Описание:**  
Плановая загрузка центров пациентами

> 🧮 **Расчёт:** `100 * План часов пациентов / Мощность`
>
> ⚠️ **Примечание:** Может присутствовать только на дашбордах с плановыми показателями

---

## ⏱️ Часы работы и мощность

### <a id="head-24"></a>🎯 План часов пациентов

**📝 Описание:**  
Общее запланированное количество часов для обслуживания пациентов

> ⚠️ **Примечание:** Может присутствовать только на дашбордах с плановыми показателями

---

### <a id="head-25"></a>🎯 План часов врачей

**📝 Описание:**  
Общее запланированное количество часов для работы врачей

> ⚠️ **Примечание:** Может присутствовать только на дашбордах с плановыми показателями

---

### <a id="head-26"></a>⏱️ Факт часов пациентов

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Часы пациентов |
| Фактическое время, ч |
| Actual Patient Hours |

**📝 Описание:**  
Количество фактически отработанных часов приёма пациентов

> 🧮 **Расчёт:** Считается по дошедшим пациентам

---

### <a id="head-27"></a>⏱️ Факт часов врачей / Выставленное время

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Выставленное время в расписании |
| Факт часов врачей |
| Часы врачей, ч |

**📝 Описание:**  
Количество часов в выставленном расписании врачей

> ⚙️ **Особенность:** В зависимости от настроек обработки данных, выставленное расписание может считаться с учётом резервов (отпуска, болезни и т.д.) или без них

---

### <a id="head-28"></a>📅 Факт часов пациентов, год назад

**📝 Описание:**  
Общее фактическое количество часов, затраченных на обслуживание пациентов, в предыдущем году за аналогичный период

---

### <a id="head-29"></a>📅 Факт часов врачей, год назад

**📝 Описание:**  
Общее фактическое количество часов, затраченных врачами на выполнение обязанностей, в предыдущем году за аналогичный период

---

### <a id="head-30"></a>⚡ Мощность, ч

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Capacity, hours |
| Максимальная доступная мощность |

**📝 Описание:**  
Максимально возможный объём работы клиники в часах

> 🧮 **Расчёт:** `# кабинетов * количество часов времени работы клиники за период`
>
> ⚙️ **Коэффициент share:** Если поддерживается справочник долей кабинетов, может учитываться коэффициент, который учитывает случаи, когда кабинет работает не весь день или распределён по нескольким направлениям, а также неврачебные кабинеты

---

### <a id="head-31"></a>⚡ Мощность, год назад

**📝 Описание:**  
Максимально возможный объём работы клиники в предыдущем году за аналогичный период

---

## 📈 Прогнозные и предельные метрики

### <a id="head-32"></a>🔝 Max # посещений кабинетов

**📝 Описание:**  
Мощность кабинетов, выраженная в максимально возможном количестве посещений

> 🧮 **Расчёт:** `Мощность, ч / Факт средняя продолжительность посещения, ч`

---

### <a id="head-33"></a>🔝 Max # посещений врачей

**📝 Описание:**  
Максимальное количество посещений, которое можно разместить в выставленном расписании врачей

> 🧮 **Расчёт:** `Количество часов в выставленном расписании / Факт средняя продолжительность посещения, ч`

---

### <a id="head-34"></a>📅 Максимум посещений 7/14/30 дней

**📝 Описание:**  
Рассчитывается как отношение суммарной длительности планового расписания врача за соответствующий период к средней длительности слота приёма

> 📌 **Примечание:** Доступно на вкладке «Заполнение расписания»

### <a id="head-35"></a>📊 Заполнение расписания на 7/14/30 дней, %

## 💰 Стоимость и эффективность

### <a id="head-50"></a>💵 Стоимость часа пациента, руб.

**📝 Описание:**  
Средняя стоимость часа работы врача на основе фактического дохода и фактически отработанных часов приёма

> 🧮 **Расчёт:** `Фактическая выручка / Факт часов пациентов`
>
> 🎯 **Интерпретация:** Показывает доходность времени, затраченного непосредственно на пациентов

---

### <a id="head-51"></a>💵 Стоимость часа врача, руб.

**📝 Описание:**  
Средняя стоимость часа расписания, выставленного на врача

> 🧮 **Расчёт:** `Фактическая выручка / Факт часов врачей`
>
> 🎯 **Интерпретация:** Показывает доходность всего выделенного врачебного времени, включая простои

---

### <a id="head-53"></a>📅 Составлено расписание на месяц, час

Сумма часов выставленного расписания за рассчитываемый месяц

> 📌 **Примечание:** Доступно на вкладке «Эффективность расписания»

**📝 Описание:**  
Разница между выставленным временем в расписании за последние 30 дней и за рассчитываемый месяц

> 🧮 **Расчёт:** `Выставленное время в расписании за последние 30 дней - Выставленное время в расписании за рассчитываемый месяц`
>
> 📌 **Примечание:** Доступно на вкладке «Эффективность расписания»

### <a id="head-55"></a>💰 Средняя совокупная выручка на амбулаторный час, руб

**📝 Описание:**  
Средняя выручка, генерируемая за один час выставленного расписания, с учётом исполненных направлений

> 🧮 **Расчёт:** `(Фактическая выручка + Выручка исполненных направлений) / Выставленное время в расписании`
>
> 📌 **Примечание:** Доступно на вкладке «Эффективность расписания»

### <a id="head-56"></a>💸 Недополученная выручка

**📝 Описание:**

>
> 📌 **Примечание:** Доступно на вкладке «Эффективность расписания»

> 💡 **Рекомендации по использованию:**
> - **Конверсия (CR)** — ключевой индикатор работы регистратуры: целевое значение зависит от специфики клиники (обычно 70–90%)
> - **Загрузка врачей** и **загрузка клиники** — разные метрики: первая показывает утилизацию времени врача, вторая — использование инфраструктуры
> - **Мощность** рассчитывается с учётом коэффициента `share` — убедитесь, что справочник кабинетов настроен корректно
> - **Прогнозные метрики (7/14/30 дней)** помогают оперативно управлять расписанием: выявлять «окна» и перераспределять нагрузку
> - **Недополученная выручка** — оценочный показатель: используйте его для приоритезации заполнения расписания, но не как точный финансовый прогноз
> - Все **плановые метрики** требуют предварительно настроенного плана в системе — без этого они не отображаются

---

> 🔐 **Требования к данным:**
> - Ведение расписания в МИС с обязательным заполнением слотов и статусов записей
> - Корректная разметка «дошедших» пациентов (галка «Пришёл»)
> - Настройка справочника кабинетов с коэффициентами долей (при расчёте мощности)
> - Интеграция данных о выручке для расчёта стоимости часа и недополученной выручки
