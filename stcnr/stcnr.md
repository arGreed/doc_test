## 📋 Базовые метрики госпитализаций

### <a id="head-1"></a>🛏️ # госпитализаций

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Количество госпитализаций |
| Hospitalizations Count |
| Admission Cases |

**📝 Описание:**  
Общее количество случаев госпитализации пациентов в стационар за выбранный период

> 📌 **Определение:** Одна госпитализация = один эпизод поступления пациента в стационар (независимо от длительности)
>
> 🎯 **Интерпретация:** Базовый индикатор нагрузки стационара и спроса на стационарные услуги

---

### <a id="head-2"></a>✅ # завершённых госпитализаций

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Завершённые случаи |
| Completed Hospitalizations |
| Discharged Cases |

**📝 Описание:**  
Количество госпитализаций, которые были завершены (пациент выписан) в выбранном периоде

> 🧮 **Критерий:** Госпитализация считается завершённой, если пациенту присвоен статус «Выписан» / «Завершено» в МИС
>
> 📌 **Важно:** Не учитываются текущие (активные) госпитализации, где пациент ещё находится в стационаре
>
> 🎯 **Интерпретация:** Позволяет анализировать только закрытые эпизоды лечения для расчёта средних значений

---

### <a id="head-3"></a>📥 # поступивших пациентов

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Принято пациентов |
| Admitted Patients |
| New Admissions |

**📝 Описание:**  
Количество уникальных пациентов, поступивших в стационар за выбранный период

> 📌 **Особенность:** Один пациент может иметь несколько госпитализаций в периоде — в этой метрике он учитывается один раз
>
> 🎯 **Интерпретация:** Показывает поток новых пациентов в стационар, важно для планирования ресурсов

---

### <a id="head-4"></a>📤 # выписанных пациентов

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Выписано пациентов |
| Discharged Patients |
| Completed Patients |

**📝 Описание:**  
Количество уникальных пациентов, выписанных из стационара за выбранный период

> 📌 **Особенность:** Один пациент может быть выписан несколько раз в периоде (повторные госпитализации) — учитывается как один уникальный пациент
>
> 🎯 **Интерпретация:** Индикатор пропускной способности стационара и завершения эпизодов лечения

---

## 📊 Эффективность и экономика стационара

### <a id="head-5"></a>📅 Койко-дни выписанных

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Среднее количество койко-дней |
| Avg Length of Stay (ALOS) |
| Средняя длительность госпитализации |

**📝 Описание:**  
Среднее количество койко-дней, проведённых пациентом в стационаре до выписки

> 🧮 **Расчёт:** `# использованных койко-дней / # выписанных пациентов`
>
> 📌 **Определение:** 
> - Койко-день = один день пребывания пациента в стационаре
> - Учитываются только завершённые госпитализации
>
> 🎯 **Интерпретация:** 
> - Высокое значение → сложные случаи, длительное лечение, возможная неэффективность оборота коек
> - Низкое значение → короткие госпитализации, дневной стационар, высокая оборачиваемость
>
> 💡 **Бенчмарк:** Зависит от профиля отделения (хирургия, терапия, реабилитация)

---

### <a id="head-6"></a>💰 Выручка на госпитализацию

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Средняя выручка на случай |
| Revenue per Hospitalization |
| Avg Case Revenue |

**📝 Описание:**  
Средняя выручка, полученная за один случай госпитализации

> 🧮 **Расчёт:** `Выручка за все госпитализации / # случаев госпитализации`
>
> 📌 **Особенность:** 
> - Учитывается выручка по всем услугам, оказанным в рамках госпитализации
> - Может включать стоимость койко-дней, процедур, медикаментов, питания
>
> 🎯 **Интерпретация:** 
> - Показывает доходность одного эпизода стационарного лечения
> - Помогает сравнивать эффективность разных отделений или программ госпитализации
> - Используется для расчёта рентабельности и планирования тарифов

## 🔄 Связь метрик: формулы и производные показатели

> 💡 Эти показатели можно комбинировать для получения дополнительных инсайтов:

| Производный показатель | Формула | Интерпретация |
|-----------------------|---------|---------------|
| **Оборачиваемость койки** | `30 / Койко-дни выписанных` | Сколько раз в месяц койка освобождается для нового пациента |
| **Загрузка коечного фонда, %** | `Факт койко-дней / (Кол-во коек × Дней в периоде) × 100` | Насколько эффективно используется инфраструктура |
| **Средняя выручка на койко-день** | `Выручка на госпитализацию / Койко-дни выписанных` | Доходность одного дня пребывания пациента |
| **Конверсия поступления в выписку** | `# выписанных / # поступивших × 100` | Доля пациентов, завершивших лечение в периоде |

---

> 💡 **Рекомендации по использованию:**
> - **Койко-дни выписанных** — ключевой показатель эффективности стационара: сравнивайте с отраслевыми бенчмарками по профилю отделения
> - **Выручка на госпитализацию** помогает оценивать рентабельность программ: если значение ниже себестоимости — требуется пересмотр тарифов или структуры услуг
> - При анализе **# завершённых госпитализаций** учитывайте сезонность: в конце месяца/квартала может быть искусственное снижение из-за незакрытых эпизодов
> - **Разница между # госпитализаций и # завершённых** показывает количество активных случаев — важно для оперативного управления ресурсами
> - Для корректного расчёта **выручки на госпитализацию** убедитесь, что все услуги в рамках эпизода правильно привязаны к случаю госпитализации в МИС

---

> 🔐 **Требования к данным:**
> - В МИС должен быть настроен модуль стационара с учётом статусов: «Поступил», «Находится», «Выписан», «Переведён»
> - Койко-дни должны фиксироваться автоматически на основе дат поступления и выписки
> - Выручка должна агрегироваться по эпизодам госпитализации (через связь «услуга → госпитализация»)
> - Для расчёта средних значений требуется достаточный объём данных (рекомендуется период ≥ 1 месяца)

---

> ⚙️ **Настройки и кастомизация:**
> - Критерий «завершённости» госпитализации может настраиваться: только «Выписан» или также «Переведён», «Выбыл»
> - Для расчёта выручки можно исключить/включить отдельные категории услуг (например, питание, медикаменты)
> - Профили отделений (хирургия, терапия, реабилитация) позволяют сегментировать анализ и сравнивать показатели внутри однородных групп
