## 🎯 Лояльность и возвращаемость

### <a id="head-1"></a>🔁 Возвращаемость к врачу в периоде, %

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Возврат к врачу, % |
| Physician Retention Rate |
| Repeat Visit Rate |

**📝 Описание:**  
Процент новых пациентов, которые вернулись к тому же врачу хотя бы ещё один раз в выбранном периоде

> 🧮 **Расчёт:** `Вернувшиеся к врачу в периоде / # Новых пациентов * 100`
>
> 🎯 **Интерпретация:** 
> - Высокий % → пациенты довольны врачом, формируется лояльность
> - Низкий % → возможная проблема с качеством приёма или коммуникацией
>
> 📌 **Примечание:** Учитываются только пациенты, чей первый визит к врачу попал в анализируемый период

---

### <a id="head-2"></a>🛤️ Путь пациента: # Посещений

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Средняя длина пути пациента |
| Avg Patient Visits |
| Visit Frequency |

**📝 Описание:**  
Среднее количество посещений пациента за выбранный период (у любых врачей клиники)

> 🧮 **Расчёт:** `Общее количество посещений / # уникальных пациентов`
>
> 🎯 **Интерпретация:** Показывает, насколько глубоко пациенты вовлечены в процессы клиники

---

### <a id="head-3"></a>❤️ Постоянные пациенты

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Лояльные пациенты |
| Регулярные пациенты |
| Repeat Patients |

**📝 Описание:**  
Пациенты, имеющие 2 и более визита в клинику

> 🎯 **Определение:** Постоянный пациент = пациент с ≥2 визитами за всю историю наблюдений в системе
>
> 📌 **Примечание:** Ключевая группа для анализа удержания и LTV

---

### <a id="head-4"></a>🚶 Разовые пациенты

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Одноразовые пациенты |
| One-time Patients |
| Single Visit Patients |

**📝 Описание:**  
Пациенты, имеющие ровно 1 визит в клинику

> 🎯 **Определение:** Разовый пациент = пациент с ровно 1 визитом за всю историю наблюдений
>
> 📌 **Примечание:** Высокая доля таких пациентов может указывать на проблемы с удержанием или специфику услуг (например, разовые консультации)

---

### <a id="head-5"></a>📊 Доля разовых, %

**📝 Описание:**  
Доля разовых пациентов от общего количества пациентов

> 🧮 **Расчёт:** `100 * Разовые пациенты / Пациенты`
>
> 🎯 **Интерпретация:** 
> - Низкий % → высокая лояльность, пациенты возвращаются
> - Высокий % → требуется работа над удержанием или анализ причин одноразовых визитов

## 👨‍⚕️ Анализ врачебного состава

### <a id="head-6"></a>👥 # врачей

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Количество врачей |
| Physician Count |
| Active Doctors |

**📝 Описание:**  
Количество врачей, имеющих оказанные услуги в выбранном периоде

> 📌 **Примечание:** Учитываются только врачи с фактической активностью (оказанные услуги), а не просто наличие в штатном расписании

---

### <a id="head-7"></a>🗂️ Группа врача ABC

**📝 Описание:**  
Классификация врачей по методу ABC-анализа в зависимости от вклада в общую выручку

| Группа | Доля в выручке | Характеристика |
|--------|----------------|----------------|
| **A** | ~80% | Ключевые врачи, приносящие основную выручку |
| **B** | ~15% | Врачи среднего вклада, стабильная загрузка |
| **C** | ~5% | Врачи с низким вкладом, начинающие или узкие специалисты |

> 🎯 **Применение:** 
> - Приоритезация мотивации и развития персонала
> - Анализ зависимости выручки от ключевых специалистов
> - Планирование найма и распределения нагрузки

---

### <a id="head-8"></a>🎯 # врачей, обеспечившие 80% выручки

**📝 Описание:**  
Количество врачей, которые в совокупности обеспечивают 80% всей выручки клиники

> 🧮 **Расчёт:** Считаются врачи с наибольшей выручкой, начиная с самого доходного, пока не будет достигнут порог 80%
>
> 🎯 **Интерпретация:** 
> - Малое число → высокая зависимость от «звёзд», риск при их уходе
> - Большое число → более равномерное распределение выручки, устойчивость

---

### <a id="head-9"></a>📊 % врачей, обеспечившие 80% выручки

**📝 Описание:**  
Доля врачей (в процентах от общего числа), которые обеспечивают 80% всей выручки

> 🧮 **Расчёт:** `(# врачей, обеспечивших 80% выручки / Общее количество врачей) * 100`
>
> 🎯 **Интерпретация:** 
> - Низкий % (например, 10–20%) → высокая концентрация выручки на ключевых специалистах
> - Высокий % (40%+) → более сбалансированная структура доходов

---

### <a id="head-10"></a>📈 Ранжирование общей выручки по сумме

**📝 Описание:**  
Присваивает ранг каждой строке результирующего набора данных по убыванию суммы выручки

> ⚙️ **Особенность:** 
> - Если строки имеют одинаковое значение выручки, им присваивается одинаковый ранг
> - Следующий ранг пропускается (стандартное плотное ранжирование)
>
> 🎯 **Применение:** 
> - Быстрое определение топ-врачей / услуг / отделений
> - Построение рейтингов и дашбордов с динамической сортировкой

---

### <a id="head-11"></a>💰 Выручка потенциальная на норму 160 часов в месяц

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Потенциал выручки врача |
| Potential Revenue @ 160h |
| Forecast Revenue per Doctor |

**📝 Описание:**  
Оценка потенциальной месячной выручки врача при загрузке на норму 160 часов

> 🧮 **Расчёт:** 
> 1. Берётся средняя выручка фактического часа врача за предыдущий месяц
> 2. Если данных за предыдущий месяц нет — берётся месяц ранее
> 3. Результат умножается на 160 часов
>
> `Средняя выручка часа врача × 160`
>
> 🎯 **Интерпретация:** Показывает финансовый потенциал врача при полной загрузке
>
> ⚠️ **Важно:** Не учитывает сезонность, специфику услуг и реальную доступность слотов

## 📅 Поведение и активность пациентов

### <a id="head-12"></a>💵 Выручка за первый визит (новые), руб

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Выручка первого визита |
| New Patient First Visit Revenue |

**📝 Описание:**  
Суммарная выручка, полученная за первый день визита нового пациента

> 🎯 **Определение:** Первый визит = дата первого оказания услуги пациенту в клинике
>
> 🎯 **Интерпретация:** 
> - Показывает эффективность «входа» пациента в клинику
> - Помогает оценить качество первичного приёма и кросс-продаж на старте

---

### <a id="head-13"></a>📆 Период активности, месяцев

**📝 Описание:**  
Количество месяцев, в которых у пациента зафиксирована выручка (оказаны услуги)

> 🧮 **Расчёт:** Подсчёт уникальных месяцев с хотя бы одной оказанной услугой за всю историю пациента
>
> 🎯 **Интерпретация:** 
> - Длинный период → пациент регулярно пользуется услугами клиники
> - Короткий период → возможно, разовое обращение или потерянный пациент

---

### <a id="head-14"></a>⏸️ Период неактивности, месяцев

**📝 Описание:**  
Количество месяцев без посещений пациента после последнего визита

> 🧮 **Расчёт:** Разница между текущим месяцем и месяцем последнего оказания услуги пациенту
>
> 🎯 **Интерпретация:** 
> - Рост периода → риск оттока пациента
> - Используется для триггеров реактивации (напоминания, акции)

---

### <a id="head-15"></a>🔄 Частота посещений, в год

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Visit Frequency per Year |
| Годовая частота визитов |

**📝 Описание:**  
Среднее количество посещений пациента в год

> 🧮 **Расчёт:** `Общее количество посещений пациента / Количество лет наблюдения`
>
> 🎯 **Интерпретация:** 
> - Высокая частота → хронические пациенты, программы ведения, ДМС
> - Низкая частота → профилактика, разовые консультации

---

### <a id="head-16"></a>📅 Дата последней записи

**📝 Описание:**  
Дата последней записи пациента в расписании клиники

> 🧮 **Расчёт:** Максимальная дата записи из расписания по пациенту
>
> 🎯 **Интерпретация:** 
> - Позволяет оценить «свежесть» контакта с пациентом
> - Используется для сегментации: активные / спящие / ушедшие пациенты

---

### <a id="head-17"></a>👨‍⚕️ Врач последнего посещения

**📝 Описание:**  
Врач, который оказывал пациенту последнюю услугу

> 🧮 **Расчёт:** Врач, связанный с последней по дате оказанной услугой пациента
>
> 🎯 **Применение:** 
> - Персонализированные коммуникации («Ваш врач Иванова ждёт вас»)
> - Анализ привязки пациента к конкретному специалисту

---

### <a id="head-18"></a>🏥 Специальность последнего посещения

**📝 Описание:**  
Специальность врача, который оказывал пациенту последнюю услугу

> 🧮 **Расчёт:** Специальность, связанная с последней по дате оказанной услугой
>
> 🎯 **Применение:** 
> - Анализ миграции пациентов между направлениями
> - Планирование кросс-направлений и комплексных программ

---

## 🩺 Первичные приёмы

### <a id="head-19"></a>🆕 # первичных приемов

**📋 Альтернативные наименования:**

| Название | 
|----------|
| Первичные консультации |
| Primary Appointments Count |

**📝 Описание:**  
Количество первичных врачебных приёмов (консультаций, осмотров), оказанных пациенту при первом обращении к специалисту

> 🧮 **Расчёт:** Подсчёт услуг, в названии которых содержится текст «первичн» (регистронезависимый поиск)
>
> ⚙️ **Настройка:** Ключевые слова для поиска настраиваются в справочнике услуг клиники
>
> 🎯 **Интерпретация:** 
> - Индикатор потока новых обращений по специальностям
> - База для расчёта конверсии «первичный → повторный»

> 💡 **Рекомендации по использованию:**
> - **Возвращаемость к врачу** — ключевой индикатор качества приёма: отслеживайте в динамике по каждому специалисту

> - **Период неактивности** используйте для настройки триггеров реактивации: например, рассылка при 3+ месяцах без визитов

> - **# первичных приемов** зависит от корректности наименований услуг в МИС: убедитесь, что ключевые слова («первичн») настроены единым стандартом

---

> 🔐 **Требования к данным:**
> - Корректная связь «пациент → услуга → врач → специальность» в МИС

> - Наличие дат записей и оказанных услуг для расчёта временных метрик
> - Настройка справочника услуг с признаком «первичный приём» или единым стандартом наименований
